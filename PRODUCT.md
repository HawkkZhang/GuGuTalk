# Product

## Register

product

## Users

Desktop Voice Input serves Chinese-speaking macOS users who write frequently in chat, documents, notes, browsers, and work tools. They use it in focused daily contexts where typing would interrupt thought, and they need voice input to feel fast, quiet, and dependable.

## Product Purpose

The product is a macOS menu bar voice input tool. It lets users press a shortcut, speak, preview recognition status, and insert the final text into the current input field. Success means the app feels like a native system companion: always available, easy to trust, and invisible when not needed.

## Brand Personality

Quiet, light, restrained. The product should feel calm and competent rather than promotional. It should carry the confidence of a small native Mac utility, with only enough visual character to make status and action clear.

## Anti-references

It should not look like a flashy AI tool, a web admin dashboard, a game panel, or a highly decorative demo. Avoid visual noise, heavy gradients, overdone glass effects, oversized marketing-style panels, and novelty interactions that make routine use feel slower.

## Design Principles

- Native first: use familiar macOS controls, spacing, materials, and system typography unless there is a clear product reason to differ.
- Calm by default: make the normal path feel quiet, compact, and stable; reserve emphasis for recording, errors, and permission blockers.
- Short paths: common actions should be visible without explanation, and advanced cloud settings should stay out of the way until needed.
- Trust through clarity: show enough state for users to understand what is happening, especially permissions, active provider, recording state, and errors.
- Respect the current workspace: never let the app's UI compete with the user's foreground task.

## Accessibility & Inclusion

The app should follow the system appearance by default and also offer manual Light and Dark choices. Text contrast should stay comfortably readable in both modes. Motion should feel smooth and premium, but remain short, state-driven, and compatible with reduced-motion expectations.
