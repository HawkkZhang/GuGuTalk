# GuGuTalk

一款 macOS 原生语音输入工具，支持本地识别和云端识别，让语音输入更高效。

## ✨ 特性

- 🎤 **多种识别引擎**
  - 本地识别（Apple Speech）：隐私优先，离线可用
  - 豆包语音识别：高质量中文识别，自动标点
  - 千问语音识别：实时流式识别，支持多语言

- ⌨️ **灵活的热键控制**
  - 按住说话模式（Hold-to-Talk）
  - 切换模式（Toggle）
  - 自定义热键组合

- 🎨 **优雅的识别气泡**
  - 实时显示识别结果
  - 流畅的动画效果
  - 自适应文本长度

- 🔧 **智能后处理**
  - 热词替换（支持正则表达式）
  - 标点符号处理
  - LLM 智能优化（可选）

## 📦 安装

### 系统要求

- macOS 13.0 (Ventura) 或更高版本
- 麦克风权限
- 辅助功能权限（用于文本插入）

### 下载

从 [Releases](https://github.com/HawkkZhang/GuGuTalk/releases) 下载最新版本。

### 首次运行

1. 打开应用后，应自动进入设置/引导窗口
2. 如果缺少权限，会直接进入权限页
3. 授予必要的权限：
   - **麦克风权限**：用于录音
   - **辅助功能权限**：用于插入文本到其他应用
   - **输入监控权限**：用于全局快捷键

如果应用只出现在菜单栏而没有打开设置窗口，说明当前构建的启动入口还需要修复；菜单栏图标里的“设置”仍可手动打开。

## 🚀 快速开始

### 使用本地识别（无需配置）

1. 打开设置 → 常用 → 识别模式：选择"本地"
2. 设置热键（默认：`⌘ + Shift + Space`）
3. 在任意应用中按住热键说话，松开后文本自动插入

**注意**：本地识别使用 Apple 设备端识别，隐私优先但无自动标点。

### 使用云端识别（需要配置）

#### 豆包语音识别

1. 获取 API 凭证：
   - 访问 [火山引擎控制台](https://console.volcengine.com/speech/service)
   - 创建应用，获取 App ID、Access Token、Resource ID
   
2. 在设置中配置：
   - 识别模式：选择"豆包"
   - 填入 App ID、Access Token、Resource ID
   - Endpoint 使用默认值即可

#### 千问语音识别

1. 获取 API Key：
   - 访问 [阿里云百炼平台](https://bailian.console.aliyun.com/)
   - 创建 API Key
   
2. 在设置中配置：
   - 识别模式：选择"千问"
   - 填入 API Key
   - Model 和 Endpoint 使用默认值

## ⚙️ 配置说明

### 识别设置

- **识别模式**：本地 / 豆包 / 千问
- **语言**：中文（简体）/ 英语 / 日语等
- **端点检测**：语音活动检测（自动断句）/ 手动（松开热键结束）

### 热键设置

- **主热键**：触发语音识别
- **切换热键**：切换识别引擎
- 支持组合键：`⌘`、`⌥`、`⌃`、`⇧`

### 后处理

- **标点模式**：保持原样 / 去掉标点 / 中文标点 / 英文标点
- **热词替换**：自定义文本替换规则
- **LLM 优化**：使用 AI 优化识别结果（需配置 LLM API）

## 🛠️ 开发打包

本项目统一把本地生成的 DMG 放在：

```bash
dist/dmg/
```

生成新的测试 DMG 时，请使用：

```bash
./scripts/package-dmg.sh
```

不要把 DMG 放到项目根目录、`Packages/`、桌面或临时目录。`dist/dmg/` 下的 DMG 和校验文件是本地构建产物，不提交到 Git。

## 🐛 已知问题

- **本地识别无标点**：Apple 设备端识别不支持自动标点，建议使用云端识别
- **权限设置**：首次使用需要授予麦克风、辅助功能、输入监控等权限
- **启动入口**：当前正在将设置页从 SwiftUI `Settings` 场景迁移为独立窗口，以确保从 Finder/Launchpad 打开应用时稳定显示设置/引导界面
- **云端配置**：需要自行申请 API 凭证

## 🔒 隐私说明

- **本地识别**：音频不离开设备，完全隐私
- **云端识别**：音频会发送到对应的云服务商（豆包/千问）进行识别
- **LLM 后处理**：如启用，识别结果会发送到配置的 LLM 服务

所有 API 凭证仅存储在本地，不会上传到任何服务器。
